**⚠️ ВАЖНО: Это студенческий прототип и экспериментальный проект**
Код предоставляется исключительно в образовательных и исследовательских целях. Он не прошел аудит безопасности и не рекомендуется для использования в production-среде или для хранения действительно критически важных данных. Мы активно разрабатываем и тестируем его, но он может содержать ошибки и недочеты.

Murnet — это высокозащищенная, децентрализованная платформа для peer-to-peer (P2P) коммуникаций, разработанная с нуля с учетом современных угроз. Проект нацелен на создание безопасного, приватного и устойчивого к цензуре протокола обмена сообщениями и данными.

**Ключевая особенность Murnet — это "Security by Design".** Безопасность встроена на всех уровнях: от криптографических примитивов до сетевого транспорта и API.

## 🚀 Ключевые возможности

*   **Полная децентрализация:** Никаких центральных серверов. Сеть основана на протоколе Kademlia (реализация `murnaked`).
*   **Hardened-криптография:**
    *   **Идентификация:** Ed25519 для подписей. Адрес генерируется из публичного ключа через `Blake2b-160` (замена устаревшего `RIPEMD160`).
    *   **Ключи:** Защита от атаки Chalkias (Double Public Key Signing Oracle Attack). Приватный ключ никогда не используется для сторонних HMAC.
    *   **Ключевая деривация:** Argon2id (победитель Password Hashing Competition) для создания ключей из паролей/сидов.
    *   **Транспортное шифрование:** Сессионные ключи на основе X25519 с аутентификацией пакетов (HMAC-Blake2b).
    *   **E2E шифрование:** AES-256-GCM для сообщений.
*   **Продвинутая защита сети:**
    *   **Защита от Replay-атак:** Нонсы (nonces) и временные метки (timestamps) во всех протокольных сообщениях (транспорт, DHT RPC, LSA).
    *   **Rate Limiting:** Пакетные и глобальные лимиты для предотвращения DDoS и флуда.
    *   **Аутентифицированная маршрутизация:** Link-State Protocol (OSPF-like) с подписанными LSA (Link State Advertisements) и системой доверия (Trust Score) для изоляции вредоносных узлов.
    *   **Circuit Breaker & Backpressure:** Защита самого узла от перегрузок.
*   **Готова к продакшену (в теории):**
    *   **Для мобильных устройств:** Адаптивные интервалы синхронизации, оптимизация батареи, batch-обработка, умное переключение между WiFi и мобильными сетями.
    *   **Для VDS:** Готовые конфигурации для Docker и systemd, интеграция с Prometheus для мониторинга.
*   **Богатый API:** Удобный REST API и WebSocket для создания приложений (чат-боты, десктопные/мобильные клиенты).

## ⚡ Быстрый старт

### Установка

```bash
# 1. Клонируйте репозиторий
git clone https://github.com/cotafei/MurNet.git
cd MurNet

# 2. Установите зависимости (рекомендуется использовать venv)
python -m venv venv
source venv/bin/activate  # Linux/Mac
# venv\Scripts\activate  # Windows

pip install -r requirements.txt
```

### Запуск узла в интерактивном режиме (CLI)

```bash
python -m murnet.cli
```

### Запуск узла как демона (фоновый режим)

```bash
python -m murnet.cli --daemon --port 8888 --data-dir ./murnet_data
```

### Отправка первого сообщения

```bash
# После запуска узла, используйте команду send
murnet> send <адрес_получателя> "Привет, Murnet!"
```
